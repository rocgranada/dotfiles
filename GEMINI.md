# Coding Standards
* **Structure:** Separation of concerns. Use of abstractions. Clear APIs using Public and Private methods. Minimize dependencies. Provide reason for new dependencies.
* **Style:** Use 4 spaces for indentation.

# Rules & Constraints
* **MUST:** Add docs for public methods. Docs explain why, not what. Always include tests. Always run the tests for the changes introduced.
* **NEVER:** Use generic exceptions. Use generic typing.

# Testing
* **Tool:** Pytest. JUnit.
* **Best practices:** Prefer Unit tests. Only test public methods. Avoid Mocks.
